# 使い方・実行手順

## アプリの起動

```bash
npm run dev
```

起動後、ターミナルに表示された URL（通常 `http://localhost:5173/monocap-vrm/`）をブラウザで開く。

## 基本的な使用手順

### Step 1: VRM モデルを用意する

- **VRoid Hub**: https://hub.vroid.com/ → `.vrm` ファイルをダウンロード
- **Booth**: https://booth.pm/ で「VRM 無料」検索

### Step 2: VRM モデルを読み込む

1. `[VRM 読み込み]` ボタンをクリック
2. ファイル選択またはサーバー上のモデルを選択
3. ステータスが「準備完了」になったら OK

### Step 3: カメラを起動してトラッキング開始

1. `[カメラ]` ボタンをクリック
2. ブラウザのカメラ許可ダイアログで「許可」を選択
3. ステータスが「トラッキング中」になったら完了

> 初回はモデルのダウンロードのため数十秒かかることがあります。

### Step 4: 動かしてみる

カメラの前に顔と上半身が映るように立つと、3D モデルが動きに追従します。

| 動き | 連動する箇所 |
|------|------------|
| 顔の向き（上下・左右・傾き） | 頭・首ボーン |
| 瞬き | blinkLeft / blinkRight 表情 |
| 口の開き・形 | aa / ih / ou / ee / oh 表情 |
| 腕の上げ下げ | 上腕・前腕ボーン |
| 体の傾き | 腰・脊椎ボーン |

### Step 5: スクリーンショットを撮る

`[スクリーンショット]` ボタンをクリックすると PNG でダウンロードします。  
背景を設定している場合は背景ごと合成して保存されます。

## リモートトラッキングモード（スマホ推奨）

MediaPipe の推定処理を PC サーバーで実行し、スマホの負荷を大幅に下げます。

### 前提

- PC とスマホが同じ Wi-Fi に接続されていること
- PC で `server/main.py` が起動していること（[setup.md](setup.md) 参照）

### 接続方法

スマホのブラウザで URL に `?server=` パラメータを追加してアクセスします。

```
http://<PCのIPアドレス>:5173/monocap-vrm/?server=ws://<PCのIPアドレス>:8000/ws
```

PCのIPアドレスの確認方法：

```bash
# Windows
ipconfig | findstr "IPv4"

# macOS / Linux
ip addr show | grep inet
```

ステータスが「リモートトラッキング中」と表示されれば接続完了です。

### ローカル・リモートの比較

| | ローカルモード | リモートモード |
|--|--------------|--------------|
| 推論の実行場所 | デバイス上（WASM） | PC サーバー（Python C++） |
| スマホの CPU 負荷 | 高い | 低い |
| バッテリー消費 | 多い | 少ない |
| 遅延 | なし | Wi-Fi 経由で ~10〜50ms |
| インターネット不要 | ✅ | サーバーに繋げれば OK |
| 追加設定 | 不要 | PC でサーバー起動が必要 |

## カスタマイズ例

### 追従の滑らかさ調整

[src/rigger.ts](../src/rigger.ts) の `LERP` 定数を変更（0〜1、小さいほど滑らか）：

```typescript
const LERP = 0.3  // デフォルト
const LERP = 0.1  // より滑らか（遅延が増える）
const LERP = 0.7  // より素早い（カクつきやすい）
```

### 検出フレームレートの変更

[src/main.ts](../src/main.ts) の `DETECT_INTERVAL` を調整：

```typescript
// モバイル: 10fps、PC: 30fps（デフォルト）
const DETECT_INTERVAL = 1000 / (isMobile ? 10 : 30)

// リモートモード利用時はサーバー負荷に合わせて調整可能
```
